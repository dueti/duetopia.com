这个博客本身就是用这套方案搭起来的。从买域名到文章上线，全程大约两个小时，其中大半时间花在等 DNS 生效上。

## 为什么写这个系列

网上的建站教程很多，但大多数有两个问题：要么太老（还在教你配 Apache），要么太跳（上来就粘贴命令，出错了不知道为什么）。

这个系列想做的是：**把每一步都解释清楚，遇到问题知道往哪查**。AI 是贯穿全程的工具，但不是黑盒——我会说明 AI 在哪里帮了忙，以及哪些地方还是得自己决定。

## 技术方案

| 组件 | 选择 | 理由 |
|---|---|---|
| 静态网站生成器 | Hugo | 构建极快，文章用 Markdown 写 |
| 主题 | PaperMod | 简洁、支持暗色模式、搜索开箱即用 |
| 代码托管 | GitHub | 免费，和 Cloudflare 集成无缝 |
| 部署 & CDN | Cloudflare Pages | 推送即自动部署，全球 CDN，免费 SSL |
| 域名管理 | Cloudflare DNS | 和 Pages 在同一个控制台，配置最省事 |

**总成本：域名年费之外，全部免费。**

## AI 能帮你做什么

说几个最实际的场景：

**装环境**：`brew install hugo` 报错了？把错误信息直接粘给 AI，通常一步解决。

**改配置**：不想读 Hugo 文档，直接告诉 AI "我想在导航栏加一个关于页面，配置怎么写"——它给你改好的片段，你复制进去。

**排查问题**：域名解析不生效、本地预览样式乱了、构建报错——这类问题贴给 AI 比 Google 快得多，因为它会结合你的具体情况回答。

**你还是需要自己做的事**：注册账号、买域名、写内容。这些 AI 替代不了。

## 开始之前

你需要准备的东西：

- 一台 Mac（Windows 也可以，命令略有不同）
- 一张可以支付美元的信用卡或 PayPal（买域名用，大约 $10/年）
- 基本的命令行使用经验（知道怎么打开终端、运行命令就够了）

不需要会编程，不需要了解服务器，不需要懂 CSS。
